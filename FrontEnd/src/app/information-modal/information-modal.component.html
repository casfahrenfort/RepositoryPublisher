Additional information

Explanation

This tool can be used to publish version control system repository snapshots to external digital object publishing systems. The data will be persistently published, meaning that a unique identifier will be assigned to it, which will be permanently resolvable through the issuing publishing system. A digital object can contain any amount or type of data, and is accompanied by metadata to clarify what data it is and to enable enhanced human and machine interaction with the data. After publishing, the identifier can be used to cite the data in your research.

For more information on digital objects, see the DOI handbook.

Requirements

To publish a repository, make sure it is publicly available through a URL. When choosing a publishing system, make sure you have an account with an API access token with that particular publisher. When publishing multiple snapshots at once, they will each be published separately to the publishing system and a final publication will be made which references the snapshot publications. This final publication can be used for citations which depend on all submitted snapshots.
